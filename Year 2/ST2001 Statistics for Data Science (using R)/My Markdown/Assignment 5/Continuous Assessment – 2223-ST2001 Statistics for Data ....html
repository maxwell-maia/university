        • Assignment 2 - Getting Started with R and R-Markdown

          Assignment 2 - Getting Started with R and R-Markdown
          Attached Files:
          •  Example Report Template.Rmd (1.591 KB)
          As an introduction to R and R-Markdown I have attached an example template file for you to "knit" together. 
          The first stage is to:
          1. Open the file in RStudio
          2. knit the document as a Word file
          Check that it runs OK and if you have any problems doing this then ask the tutors in the lab session.
          The second stage is to:
          3. Change "My Name" to yours and update your student id number
          4. Re-run the knitting of the document
          Check that the updates have worked and if not seek support as above. Whenever you make a change re-run the knitting like this, to check it has worked and you have not introduced an error. As you get more skilled you can do lots of changes and then knit the document, but when first learning it is best to knit after every change so you can identify errors more easily.
          The third stage is to:
          5. Add one more datapoint 24 to the dataset
          6. Add code to the R chunk to calculate the summary statistics (min, max, Q1, median, and Q3)
          7. Add code to the R chunk to calculate the five number summary, which uses Tukey's method to calculate the quartiles
          8. Describe the shape of the distribution of the data from the boxplot
          9. Include a new R chunk to create a histogram (Hint: hist() is the command needed for to create a histogram) and describe the shape of the distribution using it.
          10. Include a new R chunk to create a histogram with around 10 breakpoints using the help system in R.
          11. knit the final document to a Word file.

          You need to submit the Word file only to Blackboard using the link below.

        • Assignment 4 Quiz Guidance

          Assignment 4 Quiz Guidance
          We strongly recommend using only Firefox when taking the Assignment 4 quiz, as it is the most reliable browser across systems for rendering the mathematics correctly.
          • After opening the quiz, scroll down to check if the "Save and Submit" buttons is available. If not, restart the quiz. 
          • Do NOT use an iPad, iPhone or other smartphone device. Tablets are also not ideal, so if you wish to use these then you want to check they work well before the deadline. If the mathematical symbols/tables do not look like those in the lecture notes, then make sure you are using Firefox and in the unlikely event this does not resolve it then you should try another browser / device.
          • Do NOT navigate to other websites or links in Blackboard after the quiz is opened.
          • Do NOT use any browser navigation buttons (i.e. Back, Forward, Home, Refresh, etc.) during the quiz.
          • Do NOT open other browser windows or applications while taking a quiz.
          • Do NOT use the scrolling features of mouse wheels, trackpads, touchscreens or similar to navigate a quiz page. These scrolling operations can inadvertently change answers. Use the scroll bar to the side to be safe.
          • Do NOT leave the assessment page without clicking the "Save and Submit" button.
          • Do NOT click "Save and Submit" until you have completed the quiz.
          OTHER TIPS:
          • A wired connection is better for taking quizzes. Wireless connections, especially in public areas might have weaker connections and be prone to dropout, which increase the chances of losing your work.
          • For quizzes presented on multiple pages, make sure you only single-click the "Next" button to move forward. If you double-click then you may skip a question.

          If you have problems with using a Firefox browser on any system, then report this to the course coordinator. You should provide a screen snapshot of the error, and report the device, operating system and version of Firefox you are using. We may not be able to resolve the problem (so you may need to use another device), but we can warn others if it is likely to be a common issue.

          A scaffold for the analysis when the primary response variable is VO2 max is provided as both rmarkdown and html` files. You need to rerun the analysis using the V_IFT variables (i.e. `VIFT_Pre` vs `VIFT_Post`) to see if there has been any improvement on average and provide a confidence interval for the likely average improvement f players in the population of interest.
          Subjective impressions along with confidence intervals and hypothesis test will help you to answer the question.

          3.  Attach the knitted file (as word or html) using this submission link. Click on Assignment 5: One sample Inference for a Population Mean heading.
